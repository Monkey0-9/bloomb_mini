# Security Policy

## Supported Versions

We currently support the following versions of SatTrade for security updates:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability within SatTrade, please report it to our team. We take all security reports seriously and aim to address them promptly.

### How to Report

Please do NOT open a public issue on GitHub for security vulnerabilities. Instead, follow these steps:

1. **Email us**: Send a detailed report to [security@sattrade.io](mailto:security@sattrade.io).
2. **Include Details**:
   - A descriptive title.
   - A detailed description of the vulnerability.
   - Steps to reproduce the issue (proof of concept).
   - Potential impact if exploited.

### Our Response Process

1. **Acknowledgment**: We will acknowledge receipt of your report within 48 hours.
2. **Investigation**: Our security team will investigate the issue and determine its severity.
3. **Resolution**: If confirmed, we will work on a fix. We aim to release a patch or mitigation within 30 days for high-severity issues.
4. **Disclosure**: Once the fix is released, we will publicly disclose the vulnerability with credit to you (unless you prefer to remain anonymous).

## Disclosure Policy

We follow a coordinated disclosure policy. We ask that you give us reasonable time to address the vulnerability before sharing details publicly. In return, we commit to keeping you informed of our progress.

Thank you for helping keep SatTrade secure!
